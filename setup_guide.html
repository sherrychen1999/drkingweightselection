1. 整體流程　2. 建立 Google Sheet 資料庫 3. 部署 Apps Script 後端　4. 日常使用方式

你的工作流原本是：AnyDesk 連展望系統 → 常用報表管理 → 特定申報資料列印 (R1110) → 指定日期區間 → 點 EXCEL 檔 匯出。匯出後本來要手動刪欄、刪列、整理。

現在改成：把匯出的 XLSX 命名為 院區+日期區間.xlsx（例如 三樹0511-0524.xlsx）， 拖進「減重資料篩選器」就好。它會自動：

  • 用容錯規則篩出減重相關資料（含錯字、不分大小寫）
  • 顯示原始/保留/刪除筆數與刪除原因
  • 顯示每位醫師的筆數
  • 把資料自動寫入 Google Sheet 資料庫，按院區、日期區間分類

② 建立 Google Sheet 資料庫

1新建一份空白 Google Sheet
去 sheets.new 建立新試算表，命名為例如「減重申報資料庫」。
2打開 Apps Script 編輯器
在試算表上方點 擴充功能 → Apps Script。會打開一個新分頁。
3貼上後端程式碼
把編輯器裡原本的 function myFunction() { ... } 整段刪掉， 改貼「Apps_Script_後端程式碼.gs」整份內容。
按 Ctrl+S（Mac 是 ⌘+S）儲存，專案名稱可以叫「減重資料庫後端」。
4執行一次 setupSheets 建立工作表
編輯器頂部選擇函式 setupSheets，按「執行」。
第一次會跳出授權視窗，點「審核權限」→ 選擇你的 Google 帳號 → 可能看到「Google 尚未驗證這個應用程式」，這正常，點進階 → 前往「減重資料庫後端」(不安全) → 允許。
執行成功後，回到試算表會自動建立三個分頁：明細、上傳紀錄、區間摘要。
✅ 此時試算表已準備好。接下來部署 Web App 端點，讓篩選器可以寫入資料。

③ 部署 Apps Script 為 Web App

1點右上角「部署」→「新增部署作業」
2選擇類型
點齒輪圖示 → 選 網頁應用程式。
3設定執行權限
欄位設定
說明（描述）減重資料庫 v1
執行身分我（你的帳號）
誰可以存取任何人（這樣篩選器才能匿名 POST）
4按「部署」並複製 Web App URL
部署成功後會給一段網址，類似：
https://script.google.com/macros/s/AKfycb.../exec
把這段網址複製起來。
5把 URL 貼進篩選器
打開「減重資料篩選器.html」（用瀏覽器開），最上方「Google Sheet 連線設定」欄位貼上 URL， 按「儲存」。可以按「測試連線」確認綠燈亮起。
⚠️ 修改程式碼後：若以後改了 Apps Script 程式碼，要選擇 部署 → 管理部署作業 → 編輯 ✏️ → 版本：新增版本 → 部署， URL 不會變。
若選「新增部署作業」會產生新 URL，那個版本才會看到改動。
💡 如果你已經部署過舊版本：請務必用上面方式「新增版本」重新部署， 新版才會有「儀表板資料端點」（?action=aggregated）。

④ 日常使用方式

每個區間 13 個院區的 SOP

  1. 用 AnyDesk 連到展望系統，跑 R1110 匯出 EXCEL
  2. 把檔案另存為 院區+日期區間.xlsx，例如：
    • 三樹0511-0524.xlsx
    • 敦化0511-0524.xlsx
    • 松山0511-0524.xlsx（依此類推 13 個院區）
  3. 打開「減重資料篩選器.html」
  4. 把第 1 個院區的 XLSX 拖進工具，會顯示：
    • 院區 / 日期區間（從檔名自動抓）
    • 原始筆數 / 保留筆數 / 刪除筆數
    • 每位醫師的筆數
    • 已刪除的明細（含刪除原因）
  5. 檢查無誤後，按「⬆️ 上傳到 Google Sheet」
  6. 右上角會顯示「○ / 13（區間 XXXX-XXXX）」的進度
  7. 重複 1-6 共 13 次，完成一個區間

到 HTML 看儀表板

13 個院區傳完後，篩選器頁面下方「📊 區間儀表板」會自動顯示：

  • 各院區總週數：本期 vs 上期，含變化量與百分比
  • 各院區×醫師週數明細：每位醫師在每個院區的週數，含上期比較
  • 醫師跨院區總計：每位醫師在所有院區的週數總和

上期 = 在資料庫裡比本期早的最近一個區間（依字母順序）。可以從下拉選單切換不同區間。

「週數」怎麼算？從 labeno 抽出數字+W 的部分，例如： 減1W = 1 週、減拿4W = 4 週、M減5W.8 = 5 週。 多筆同病患算多筆，因為一筆代表一次取藥。

到 Google Sheet 看結果

分頁內容
明細所有保留資料逐筆，加上院區、日期區間欄
上傳紀錄每次上傳的檔名、原始/保留/刪除筆數、刪除明細
區間摘要每個區間×院區的檔案數、保留數、刪除數總計
同檔名會擋下：如果同一個 院區+日期區間+檔名 重複上傳， 系統會回「此檔案已上傳過」並拒絕。要重新上傳請先到 Sheet 手動刪除舊資料。

常見錯誤訊息

訊息原因 / 處理
檔名無法完整解析檔名沒有 院區0511-0524 這種格式。改名再試
找不到 labeno 欄位檔案不是 R1110 匯出的格式。請確認從展望「特定申報資料列印」匯出
連線失敗Web App URL 沒設定，或 Apps Script 已停用，或網路問題
此檔案已上傳過同檔名已寫入 Sheet。要覆寫請先到 Sheet 刪掉舊資料

📌 容錯規則小提醒

櫃檯打資料偶爾會把「減」打成「檢」，或大小寫不一致，本工具都會自動辨識：

  • 減 可寫成：減 / 檢 / 簡 / 鑑
  • 拿 可寫成：拿 / 那 / 哪 / 納 / 拎
  • 初 可寫成：初 / 出 / 除 / 楚
  • 電 可寫成：電 / 店 / 點 / 殿
  • 寄 可寫成：寄 / 既 / 計 / 季 / 紀 / 即
  • V / O / M 大小寫都可
  • 週數 1~20、後綴 轉 / .X / .P / .U / .G / .digit 都可加
  • 結尾若為 T00（例如 減免T00、減重T00）會排除

若發現某個應該保留的字眼被刪掉，請在篩選器的「已刪除明細」區塊截圖回報， 我把字加進規則庫。
